# `/config` — 可版本化的配置资产（迁移闭包的一部分）

本目录存放**非机密**的项目级配置，纳入 git、随仓库迁移（方案第十四章）。
它和 `/data`（机密 + 运行时状态）共同构成单机迁移的最小闭包。

| 文件 | 作用 | 对应章节 |
|---|---|---|
| `style-guide.yaml` | 设计系统（色板/字体/间距/圆角/动效），风格引导核心资产 | 十九 |
| `acceptance.yaml` | 验收基准（视觉比对分层打分/性能/响应式/无障碍） | 十三、十七 |
| `model-profiles.yaml` | 模型画像、路由器规则、手工指定示例、进化池配置 | 十、十一 |
| `team.yaml` | 专家团队角色定义与协作拓扑 | 十六 |

## 约定
- **只放非密**：密钥（API Key / DB 密码 / 私钥）一律走 `.env` / docker secrets，**不在此目录**。
- **环境覆盖**：`config/local*.yaml` 已被 `.gitignore` 屏蔽，本地调参用本地覆盖文件。
- **加载**：orchestrator 启动时读取本目录（见 `orchestrator/src/style_loader.py`）。
- **迁移**：换机器时 `git clone` 拉取本目录 + `rsync /data` 即可恢复全量配置与状态。

## 风格引导工作流（方案第十九章）
1. **阶段 0（一次性重投入）**：与 AI 共建 `style-guide.yaml` 与参考截图，把你的风格钉死。
2. **阶段 1（日常）**：描述需求 → Agent 自动加载本目录 → 按你的风格自主开发 + 验收。
3. **阶段 2（可选）**：上 7B 进化池让权重内化风格（开启 `model-profiles.yaml` 的 `evolution`）。
